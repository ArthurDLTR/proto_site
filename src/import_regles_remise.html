Protocole pour l'import des règles de remise dans Dolibarr

Pour utiliser les règles de remise fixées selon les catégories Client au lieu du dernier prix client, il faut décocher les 3 types de documents sur lesquels le module va chercher le prix à appliquer dans les paramètres du module Règles de remise comme montré ci-dessous. Une fois les 3 décochés, le module appliquera uniquement les règles selon catégories Client peu importe leur niveau de priorité.

Import depuis onglet dédié dans Dolibarr

Pour l'import des règles de remise, il faut le faire depuis l'onglet dédié dans la page des Produits | Services il permet également d'éviter les erreurs de saisies pour les tags Produits et Clients, un peu plus instictinf.

Il faut rentrer la quantité minimum, la remise en %, la catégorie Client et la catégorie Produit lors de la création de la règle de remise. Le reste des informations n'est pas nécessaire pour le moment. Pour les niveaux de priorités, plus le niveau est élevé plus la règle est prioritaire. En cas de règle de même niveau, la plus avantageuse pour le client est choisie.

Import via fichier CSV importé dans Dolibarr

Pour l'utilisation de l'outil d'import, il ne faut remplir que le libellé, les catégories Produit, les catégories Client, la remise en pourcentage, la quantité minimum et l'activation qui doit être à 1. Les catégories (Produit ou Client) doivent être séparées par une virgule sans espace (ex: Restaurant,Hotel). Attention, le tag doit être écrit comme l'intitulé du tag (sauf pour les capitales).
